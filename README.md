# friendcast v0.1

## 1. friendcastの概要
friendcastは「親しい人にだけ届ける、声のタイムライン」をコンセプトにしたクローズド型音声SNSです。スマホで直感的に使えるX/Twitter風タイムラインをベースにしつつ、音声投稿が主役になるUIを目指しています。

## 2. v0.1で実装した内容
- スマホファースト・ダークモードのUIプロトタイプ
- 7画面: ホーム / 投稿作成 / 公開範囲選択（投稿作成内） / 投稿詳細 / プロフィール / 友人検索・招待 / 設定
- モックデータによる複数タイプ投稿（短文のみ、音声付き、日記、愚痴、問題提起、公開範囲別、音声返信付き）
- 簡易動作:
  - 下部ナビゲーションで画面切り替え
  - 140文字カウント
  - 公開範囲選択の反映
  - 再生中っぽい見た目切り替え
  - 録音中っぽいアニメーション切り替え
  - 返信ボタンから投稿詳細へ遷移

## 3. v0.1でまだ実装していない内容
- Googleログイン
- Supabase/DB接続
- 実音声ファイルの保存・配信
- 本格的な公開範囲権限制御
- フォロー/フォロワーの実データ管理
- 通知・解析・AI機能

## 4. 次に実装すべきv0.2内容
- Googleログイン導入
- ユーザープロフィールの永続化
- フォロー/フォロワーのデータモデル実装
- 検索・招待導線のデータ接続
- 投稿の公開範囲モデル定義（UIは既存を流用）

## 5. ローカルでの起動方法
```bash
npm install
npm run dev
```
ブラウザで表示されたローカルURLにアクセスしてください。

## 6. v0.1.1 UI/UXブラッシュアップ
- ホーム投稿カードの視認性を強化（再生中の強調、公開範囲/到達先表示、保存UI追加）
- 投稿カード・投稿詳細・プロフィール履歴に絶対日時（例: 2026年5月23日 01:28）を表示
- 投稿作成画面を改善（文章/音声の導線、録音ボタン訴求、公開範囲の説明付き選択、投稿前の到達確認文）
- 公開範囲UIを説明付きカード化し、選択状態を強調
- プロフィール画面を「声のプロフィール」として強化（固定自己紹介音声 + 過去投稿タイムラインの分離表示）
- 設定画面にテーマ設定を追加（ダーク / ライト / システム）
- ダーク/ライト両テーマで余白・ガラス風カード・グラデーションを再調整

## 7. v0.1.2 X風UI/UX改善
- ホームタイムラインをX/Twitterに近い投稿レイアウトへ更新（アイコン、表示名/ID/日時の横並び、区切り線ベース）
- 投稿アクション行を追加（コメント/リポスト/お気に入り/保存/共有）
- お気に入り・保存はトグル挙動を実装、コメントは投稿詳細へ遷移
- インプレッション表示を排除し、数字競争ではなく安心感中心のUIに調整
- 公開範囲ラベルと「この声は〇〇に届きます」表示を控えめに維持
- プロフィール画面をX風に強化（ヘッダー画像、丸アイコン、自己紹介、フォロー情報、タブ）
- 「投稿/返信/音声/保存」タブ追加と、プロフィール上でのタイムライン表示統一
- 固定自己紹介音声と「自己紹介を録る」「声で投稿」導線を追加（投稿作成画面へ遷移）
- ダーク/ライトテーマの見え方を維持したまま視認性を再調整
